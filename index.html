F1 Grand Prix

F1 Grand Prix
2022

×

The 'Formule 1 Grand Prix' is an AI competition organised by the ENSICAEN in 2022. In team of two, the participants must create the best AI capable of finishing any race the fastest. The competition is not only about pathfinding, the pilot must take into account its gas consuption to be able to finish the race. There are also several obstacle on the way such as sand pit, walls and even others player that can block the progression of each other.

Here is a footage of our best racer (The Goose) competing against another participant of the competition

The language imposed for this project is the C. This project was an excellent occasion to master this language and learn about several kind of pathfinding algorithm. In the end we decided to use the A Start algorithm. Coupled with a complex graph and a heuristic using a simple yet effective heat map, our racer won the majority of the races.

During this project I learned a lot about:

  • AI algorithm and path finding
  • C language, memory manipulation and project debugging
  • A little bit of hacking: I loved tweaking the game and forcing parameters like the change of my pilot for a Goose
  • Team work within a small student team

You can find the project on my Github and also a short report in French I wrote about the solution we provided

QGIS Assisted-MapMatching

QGIS Assisted-MapMatching
2021

×

QGIS Assisted-MapMatching is a QGIS plugin to semi-automatically adjust a GPS track on a street network (map-matching). I developed it during the internship I did at the INRS (Institut national de la recherche scientifique) in Canada to complete my 2 year technical Diploma. The plugin was published on the official website of QGIS the 17th of June 2021 and has been download by more than 5000 users.

The Map-Match Plugin offers a practical solution for efficiently aligning GPS tracks with street networks within QGIS. This tool simplifies the map-matching process, eliminating the need for manual point-by-point adjustments. Here's what the plugin can do:

  1. Data Preparation: Ensure accurate results by correcting network topology and validating input data. The plugin prepares data for reliable map-matching.
  2. Network Reduction: Reduce network complexity by buffering GPS trace points. The plugin addresses topology issues to enhance data integrity.
  3. Map Matching: Choose from map-matching algorithms like by speed, by distance, or to the closest road. The plugin automates this process, with manual adjustment options available.
  4. Flexible Export: Easily export matched tracks, path polylines, or the entire project in the desired format.

This project taught me about many aspect of large scale application developement that I have never experienced before. I learned about:

  • The importance of writing clear code through the respect of strict guidelines (PEP8, SOLID principle, GRASP model)
  • How to write unit tests and document the code to create a robust program
  • The use of application like Docker to create special container capable of testing the deployment of the plugin on different platforms
  • The language Python as well as the Framework QT

Shadow Hunter

Shadow hunter
2020 - 2023

×

The 'Shadow Hunter' is a Minecraft plugin I developed in cooperation with the youtuber Jejeplusfaim. The aim of this project was to teach him the basics of java programmation through the creation of a complete plugin monitoring every aspect of the event.

The game revolve around three faction: The Shadows, The Hunters, and the Neutrals. As in the board game "Werewolf", each players own a unique character unknown to the others. Each of the 18 character has a special ability attached to him as well as a personnal victory condition and a team winning condition. In order for a player to win the game, he must fullfill at least one personnal conditions and one team condition before the other players. For the Hunters and the Shadows, these condition ofently revolve around killing rivals from the enemy camp. For the Neutral, each character has their own specific gameplay mechanics leading them toward their goal which make them potential allies or foes. The game is divided in 6 phases where players will need to find ressources on the map to craft powerfull weapons and armor, find potential allies, and slowly converge toward the center of the map for the last affrontement. Each phases is 20 minutes long and the identity of 2 random players will be revealed when going to the next phase.

Here is a quick video presenting the rule of the game and a few characters. As the video is in French, don't hesitate to contact me if you have any question about this game. I will be more than happy to share some details with you about this project

Even if my work was to teach the basics of programmation to my colleague, I still learned a lot about video game programming and how to use the minecraft Bukkit API. Here is a few things I mastered through this project:

  • The creation of a program easy to maintain and opened to expansion
  • The optimisation of gameplay mechanics to fit a low preformance device
  • The creation of a solid architecture which can resist errors and allow flexibility for the administrator during the game
  • How to write clean a easy to understand code
  • How to learn from players feedback and adjust the game to maximise the fun

I had a lot of fun going through every gameplay mechanics different between each characters. I hope to further develop this project in the future or even work on a project of the same scale to keep experiencing with the Bukkit API.
