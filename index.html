Math and Physics for video games

Math and Physics for video games
2023

×

During my first semester at the UQAC I had the opportunity to work on a project in cooperation with another student. This project aimed to build a physics engine with 2 principal systems, a particle simulation and a rigidbody simulation. To showcase the engine capabilities we also developed a basic graphic engine in Vulkan.

Here is a quick overview of the two system's capabilities:
Particle simulation:

  • Integrator for managing positions, velocities, and accelerations.
  • Force generation handling gravity, friction, and springs.
  • Collision detection system covering interpenetrations, rods, cables, and resting contacts.
  • Collision resolution using impulses.

Rigidbody simulation:

  • Extended integrator to manage rotational physics in addition to previous aspects.
  • More complex collision detection using a spatial partitioning system (grid) for the broad phase and a narrow phase to generate contact data (interpenetration, contact normal ...) on sphere, plane and boxes.
  • More complex collision resolution using rotational physics and collision form data.

You can see in the video below a short demonstration of the engine:

This project taught me about many concepts of video game development that I have never experienced before. As an example, for the implementation of the graphic engine, I learned about:

  • The basics of Vulkan
  • The implementation of a simple render pipeline in Vulkan
  • Simple shader writing in HLSL
  • The use of swap chains to draw on unused frame and then display it to the user
  • The basics of 3D rendering (Perspective projection transformation, frustum culling ...)

For the physic engine, I learned about:

  • Several mathematics principles (relation between position, velocity, acceleration: integration, collision resolution, Quaternion operations ...)
  • The way to implement any mathematical formula into my program and optimize the code for it to run smoothly
  • Simple collision detection algorithm and when to use them to get the best performances
  • How to quickly implement functionalities for showcase and then refactoring it to allow future update

QGIS Assisted-MapMatching

QGIS Assisted-MapMatching
2021

×

QGIS Assisted-MapMatching is a QGIS plugin to semi-automatically adjust a GPS track on a street network (map-matching). I developed it during the internship I did at the INRS (Institut national de la recherche scientifique) in Canada to complete my 2 year technical Diploma at the UGA. The plugin was published on the official website of QGIS the 17th of June 2021 and has been downloaded by more than 5000 users.

The Map-Match Plugin offers a practical solution for efficiently aligning GPS tracks with street networks within QGIS. This tool simplifies the map-matching process, eliminating the need for manual point-by-point adjustments. Here's what the plugin can do:

  1. Data Preparation: Ensure accurate results by correcting network topology and validating input data. The plugin prepares data for reliable map-matching.
  2. Network Reduction: Reduce network complexity by buffering GPS trace points. The plugin addresses topology issues to enhance data integrity.
  3. Map Matching: Choose from map-matching algorithms like by speed, by distance, or to the closest road. The plugin automates this process, with manual adjustment options available.
  4. Flexible Export: Easily export matched tracks, path polylines, or the entire project in the desired format.

This project taught me about many aspects of large scale application development that I have never experienced before. I learned about:

  • The importance of writing clear code through the respect of strict guidelines (PEP8, SOLID principle, GRASP model)
  • How to write unit tests and document the code to create a robust program
  • The use of application like Docker to create a special container capable of testing the deployment of the plugin on different platforms
  • The language Python as well as the Framework QT

Shadow Hunter

Shadow hunter
2020 - 2023

×

The 'Shadow Hunter' is a Minecraft plugin I developed in cooperation with the youtuber Jejeplusfaim. The aim of this project was to teach him the basics of java programmation through the creation of a complete plugin monitoring every aspect of the event.

The game revolves around three factions: The Shadows, The Hunters, and the Neutrals. As in the board game "Werewolf", each player owns a unique character unknown to the others. Each of the 18 characters has a special ability attached to him as well as a personal victory condition and a team winning condition. To win the game, a player must fulfill at least one personal condition and one team condition before the other players. For the Hunters and the Shadows, these conditions oftenly revolve around killing rivals from the enemy camp. For the Neutral, each character has their own specific gameplay mechanics leading them toward their goal which makes them potential allies or foes. The game is divided in 6 phases where players will need to find resources on the map to craft powerful weapons and armor, find potential allies, and slowly converge toward the center of the map for the last confrontation. Each phase is 20 minutes long and the identity of 2 random players will be revealed when going to the next phase.

Here is a quick video presenting the rule of the game and a few characters. As the video is in French, don't hesitate to contact me if you have any question about this game. I will be more than happy to share some details with you about this project

Even if my work was to teach the basics of programmation to my colleague, I still learned a lot about video game programming and how to use the minecraft Bukkit API. Here are a few things I mastered through this project:

  • The creation of a program easy to maintain and opened to expansion
  • The optimisation of gameplay mechanics to fit a low performance device
  • The creation of a solid architecture which can resist errors and allow flexibility for the administrator during the game
  • How to write clean a easy to understand code
  • How to learn from players feedback and adjust the game to maximise the fun

I had a lot of fun going through every gameplay mechanics different between each character. I hope to further develop this project in the future or even work on a project of the same scale to keep experiencing with the Bukkit API.
